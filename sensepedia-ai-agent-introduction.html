从回答问题到完成任务
现代 AI Agent 与 SensePedia 的智能体

像 Claude Code、Codex 那样，给一个目标，它就自己拆解、调工具、看结果、改计划，直到把事情做完。本文讲清楚什么是现代 AI Agent、有哪些范式，以及 SensePedia 平台实际用了哪几种。

阅读前的一个提示：「Agent」没有唯一官方定义。本文采用大模型时代的工程范式口径（增强型 LLM、工具调用、ReAct、规划-执行、多 Agent 等），SensePedia 部分的判定均来自对其真实代码的勘察，而非概念套用。
1Definition

什么是现代 AI Agent？

普通聊天机器人是「问一句、答一句」；现代 AI Agent 是「给一个目标，它围绕目标持续行动」。当你对编程 Agent 说「帮我加一个登录功能并保证测试通过」，它不会只吐一段代码，而是读项目 → 制定计划 → 改代码 → 跑测试 → 看到报错 → 自己修 → 再测 → 交付。

🧩 一个工程化的定义

AI Agent = 大模型 + 目标 + 环境感知 + 记忆/状态 + 决策 + 工具/行动 + 反馈循环

它不是「更聪明的模型」这么简单，而是软件形态的变化：从「调用一个模型」变成「构造一个能感知、计划、行动、反馈的系统」。

✅ 判断它是不是 Agent

满足越多，越像真正的 Agent：

有明确目标能感知环境动态决定下一步调用工具行动保持状态/记忆多步执行

一句话：Chatbot 回答问题，Workflow 按固定流程处理任务，Agent 围绕目标动态决定下一步，并通过工具影响外部环境。
2The ReAct Loop

Agent 的工作循环：ReAct

现代 Agent 的核心是 ReAct（Reasoning + Acting）：推理与行动交替进行，不是一次想完所有步骤，也不是盲目执行，而是边想、边做、边观察、边修正。

1
理解目标

把用户的一句话变成可执行任务。

2
感知环境

读文件、查数据库、搜网页、看代码仓库。

3
思考下一步

动态决定先做什么、用哪个工具。

4
调用工具行动

改文件、运行命令、调 API、检索知识。

5
观察反馈

看测试结果、报错、返回值。

6
修正并继续

不对就改计划，循环回第 3 步，直到完成。

3Paradigms

现代 Agent 有哪些范式？

大模型时代的 Agent 工程范式可以叠加——一个真实 Agent 往往同时是好几种。下面十种由浅入深：

① 增强型 LLM

给模型接上检索、工具、记忆、外部 API，是一切 Agent 的基础构件。

Augmented LLM · 底座

② 固定工作流

路径写死：分类→检索→生成→审核。稳定可控，但不算完全自治。

Workflow · 提示链/路由/并行

③ 工具调用 Agent

模型自主选择并调用工具：搜索、计算、数据库、终端、API。

Tool-Using

④ ReAct Agent

思考与行动交替，边做边观察边修正。现代 Agent 的核心思路。

Reasoning + Acting

⑤ 规划—执行

先把大任务拆成小步骤，再逐步执行，必要时重新规划。

Planner-Executor

⑥ 编排者—工人

中心 Agent 当「经理」，动态拆任务、派给多个 worker、再汇总。

Orchestrator-Workers

⑦ 多 Agent 系统

多个角色 Agent 分工协作：研究、写码、测试、审查各司其职。

Multi-Agent

⑧ 反思 / 自我改进

生成→自评→找缺陷→修改→再评；真实系统还会配外部校验。

Reflection

⑨ 记忆型 Agent

跨会话/任务保留上下文、偏好、历史与技能。

Memory-Based

⑩ 具身 / 电脑使用

在操作系统、浏览器、代码仓库等环境里直接行动。

Computer-Use

能力等级可分 L0–L5：L0 普通模型 → L1 工具增强 → L2 工作流 → L3 半自治 Agent（Claude Code / Codex 在此） → L4 多 Agent 系统 → L5 长期自治。今天可用系统主要在 L1–L4。

5Architecture

整体架构：四层叠加

SensePedia 不是单一范式，而是把多种范式叠成一套平台（能力等级 L3 半自治 ~ L4 多 Agent）。

🧭 它属于什么

大模型时代的工程范式复合体：增强型 LLM 底座 + 工具调用 + ReAct + 规划-执行 + 记忆 + 编排者-工人 + 多 Agent，并夹杂固定工作流。

🚫 它不属于什么

不属于经典 AI 范式：没有强化学习训练循环，不是 BDI（信念-愿望-意图），也不是纯反应式。它的「决策」来自大模型推理 + 工具反馈。

∑Summary

总结

🟢 自治 Agent

HostOps · Cyber · Knowledge。工具调用 + ReAct + 规划-执行 + 记忆，最像 Claude Code / Codex，是平台主体。

🔵 Worker / 构件

Intent Agent · RAG Query。介于工具与 Agent 之间的受控专职单元。

🟡 固定工作流

Customer Agent · Data Analyst · Knowledge Ingestion。路径写死，更像「AI 工作流」。

一句话定性：SensePedia 用到的 Agent，以「工具调用 + ReAct + 规划-执行 + 记忆型」的半自治 Agent 为核心，外层由「编排者-工人 / 多 Agent 系统」统筹，并夹杂若干「固定工作流」，全部建立在「增强型 LLM」底座之上。它正好印证：真实工程 Agent 往往是多种范式的叠加，而非某一种纯范式。
